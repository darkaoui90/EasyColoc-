# EasyColoc

EasyColoc is a Laravel web application to manage shared housing expenses.

It helps roommates track common expenses, compute balances automatically, and show a simplified "who owes who" view without manual calculations.

## Project Overview

This project was built as an MVC monolith using Laravel.

Main objectives:
- Manage shared houses (colocations) and members
- Track shared expenses
- Compute individual balances automatically
- Display simplified reimbursements
- Add basic platform administration and moderation

## Current Features

### Authentication and User Management
- Register, login, logout, profile management
- First registered user is automatically promoted to global admin
- Banned users are blocked by middleware and automatically logged out

### Colocations
- Create a colocation (creator becomes owner)
- List active colocations for current user
- View colocation details (members, balances, expenses, settlements)
- Leave colocation (member only, owner cannot leave)
- Cancel colocation (owner only)
- Rule enforced: one active colocation per user

### Invitations
- Owner can invite by email
- Token-based invitation links
- Invitation can be accepted or declined
- Validation checks:
  - invitation email must match logged in user email
  - invitation must not be expired
  - invitation must not already be handled
  - target user must not already have an active colocation

### Expenses and Categories
- Add expense with title, amount, date, payer
- Use existing category or create a new one directly from expense form
- Delete expense (owner or payer only)
- Monthly expense filter in colocation details

### Balances and Settlements
- Automatic balance calculation per active member
- Simplified settlements generated by matching debtors and creditors
- "Mark as paid" action for pending settlement lines
- Validation for settlement amount and permissions

### Reputation
- Reputation is updated when leaving/canceling:
  - `-1` if member has debt
  - `+1` otherwise
- Reputation displayed in layout, dashboard, and admin view

### Global Admin
- Admin dashboard with global stats:
  - total users
  - active colocations
  - total expenses
  - banned users
- User search (name/email)
- Ban/unban non-admin users
- Self-ban and admin-ban are blocked

## Roles

- `member`: regular colocation member
- `owner`: colocation owner (management permissions in own colocation)
- `admin`: global platform admin (stats and moderation)

A global admin can still act as owner/member inside colocations.

## Business Rules Implemented

- One active colocation per user
- Only active members can access and operate inside a colocation
- Only owner can invite/cancel colocation
- Only owner or payer can delete an expense
- Only debtor or owner can mark settlement as paid
- Banned users cannot use the authenticated app

## Tech Stack

- Laravel 11 (PHP 8.2+)
- Blade templates + Tailwind CSS
- MySQL (default local setup)
- Eloquent ORM
- Pest / PHPUnit for tests

## Architecture

Monolithic MVC:
- `app/Models` for domain entities
- `app/Http/Controllers` for application logic
- `app/Services/ColocationBalanceService.php` for debt simplification logic
- `resources/views` for UI
- `routes/web.php` for web routes

## Main Domain Entities

- `User`
- `Colocation`
- `Category`
- `Expense`
- `Settlement`
- `Invitation`
- Pivot: `colocation_user` (role, joined_at, left_at)

## Local Installation

### Requirements
- PHP 8.2+
- Composer
- Node.js + npm
- MySQL

### Steps

1. Clone project
```bash
git clone <your-repo-url>
cd EasyColoc
```

2. Install dependencies
```bash
composer install
npm install
```

3. Configure environment
```bash
cp .env.example .env
php artisan key:generate
```

4. Configure DB settings in `.env` then run migrations
```bash
php artisan migrate
```

5. Build frontend assets
```bash
npm run build
```

6. Start server
```bash
php artisan serve
```

Open: `http://127.0.0.1:8000`
